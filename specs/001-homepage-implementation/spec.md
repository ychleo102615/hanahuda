# Feature Specification: 花牌遊戲網站首頁

**Feature Branch**: `001-homepage-implementation`
**Created**: 2025-10-22
**Status**: Draft
**Input**: User description: "實作花牌遊戲網站的首頁。參考 @doc/readme.md 2.1.1 首頁的說明。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 快速了解遊戲並開始體驗 (Priority: P1)

新手玩家首次訪問網站時，能夠立即理解這是一個什麼樣的遊戲，並在視覺吸引下快速開始第一次遊戲體驗。

**Why this priority**: 這是網站的核心價值主張，決定了使用者是否會繼續使用產品。Hero Section 和主要 CTA 是使用者第一眼看到的內容，必須在 5 秒內傳達遊戲核心價值。

**Independent Test**: 可以透過 A/B 測試不同的 Hero Section 設計，測量「首頁到遊戲頁面」的轉換率，以及使用者在首頁的停留時間。

**Acceptance Scenarios**:

1. **Given** 使用者首次訪問網站，**When** 頁面載入完成，**Then** 使用者能在 5 秒內看到遊戲標題、副標題和「開始遊戲」按鈕
2. **Given** 使用者查看 Hero Section，**When** 使用者點擊「開始遊戲」按鈕，**Then** 系統導航至遊戲頁面
3. **Given** 使用者在首頁瀏覽，**When** 使用者滾動頁面，**Then** 視覺設計應融合傳統花牌意象與現代設計風格，提供沉浸式文化體驗

---

### User Story 2 - 學習遊戲規則 (Priority: P2)

對花牌遊戲不熟悉的國際玩家能夠透過清晰的規則說明快速理解遊戲玩法，包括牌組構成、牌的分類、基本役種和遊戲流程。

**Why this priority**: 花牌在國際市場的認知度較低，新手玩家需要完整的規則介紹才能開始遊戲。這是降低學習曲線的關鍵功能。

**Independent Test**: 可以透過使用者測試，測量新手玩家在閱讀規則後能否正確回答「遊戲目標」、「牌組構成」等基本問題。測試指標包括規則區塊的展開率和停留時間。

**Acceptance Scenarios**:

1. **Given** 使用者需要了解遊戲規則，**When** 使用者滾動到規則介紹區，**Then** 應顯示可折疊/展開的規則說明區塊
2. **Given** 使用者閱讀規則，**When** 使用者展開規則說明，**Then** 應以英文顯示以下內容：
   - 遊戲目標
   - 牌組構成（12個月份，每月4張，共48張）
   - 牌的分類（光札/Hikari、種札/Tane、短冊/Tanzaku、カス/Kasu）
   - 基本役種介紹（至少包含：五光、四光、赤短、青短、三光、花見酒、月見酒）
   - 遊戲流程說明
3. **Given** 使用者初次查看規則區塊，**When** 預設狀態下，**Then** 規則說明應為折疊狀態，使用者可點擊展開查看詳細內容
4. **Given** 使用者透過導航列點擊「規則」連結，**When** 頁面滾動至規則區塊，**Then** 規則區塊應自動展開

---

### User Story 3 - 瀏覽網站與獲取資訊 (Priority: P3)

使用者能夠透過導航列快速存取網站的不同區塊，包括規則說明、關於頁面和開始遊戲功能。

**Why this priority**: 提供基本的網站導航功能，改善使用者體驗。雖然重要，但相較於 Hero Section 和規則說明，優先級較低。

**Independent Test**: 可以透過點擊追蹤測量各導航連結的點擊率，以及使用者在不同區塊之間的導航路徑。

**Acceptance Scenarios**:

1. **Given** 使用者在首頁任意位置，**When** 使用者查看頁面頂部，**Then** 應顯示導航列，包含 Logo/遊戲名稱和導航連結
2. **Given** 使用者點擊導航列的「規則」連結，**When** 系統執行導航，**Then** 頁面應平滑滾動至規則介紹區並展開規則說明
3. **Given** 使用者點擊導航列的「關於」連結，**When** 系統執行導航，**Then** 頁面應平滑滾動至關於區塊
4. **Given** 使用者點擊導航列的「開始遊戲」連結，**When** 系統執行導航，**Then** 應跳轉至遊戲頁面

---

### User Story 4 - 查看版權與授權資訊 (Priority: P3)

使用者能夠透明地了解網站使用的美術資源來源和授權資訊，建立信任感。

**Why this priority**: 符合開源專案的最佳實踐和法律要求，但對核心遊戲體驗影響較小。

**Independent Test**: 可以透過使用者調查測量對版權聲明區塊的滿意度，以及是否影響使用者對網站的信任感。

**Acceptance Scenarios**:

1. **Given** 使用者滾動至頁面底部，**When** 使用者查看版權聲明區，**Then** 應顯示使用的花牌圖像來源聲明
2. **Given** 使用者查看版權聲明，**When** 使用者閱讀內容，**Then** 應包含開源授權資訊（如 MIT License, Public Domain）和第三方資源 attribution
3. **Given** 使用者對某個資源的授權有疑問，**When** 使用者點擊相關連結，**Then** 應能導航至原始授權頁面或資源來源

---

### Edge Cases

- **當規則說明內容過長時**：系統應在展開狀態提供內部滾動，避免佔據過多頁面空間
- **當使用者使用小螢幕設備（手機）訪問時**：導航列應能折疊為 hamburger menu，避免介面擁擠
- **當使用者瀏覽器禁用 JavaScript 時**：Hero Section 和規則說明應能正常顯示靜態內容，確保基本可讀性
- **當圖片資源載入失敗時**：應顯示替代文字和 fallback 背景色，避免頁面破版
- **當使用者快速點擊多次「開始遊戲」按鈕時**：系統應防止重複導航，確保只執行一次跳轉
- **當使用者使用鍵盤導航（無障礙需求）時**：所有互動元素應能透過 Tab 鍵和 Enter 鍵操作

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統必須在首頁顯示 Hero Section，包含遊戲標題、副標題和「開始遊戲」主要 CTA 按鈕
- **FR-002**: 系統必須在 Hero Section 使用傳統花牌意象融合現代設計風格的視覺設計
- **FR-003**: 使用者必須能夠點擊「開始遊戲」按鈕導航至遊戲頁面
- **FR-004**: 系統必須提供規則介紹區，以英文顯示 Koi-Koi 基本規則說明
- **FR-005**: 系統必須在規則介紹區包含以下內容：
  - 遊戲目標
  - 牌組構成（12個月份，每月4張，共48張）
  - 牌的分類（光札/Hikari、種札/Tane、短冊/Tanzaku、カス/Kasu）
  - 基本役種介紹（至少包含：五光、四光、赤短、青短、三光、花見酒、月見酒）
  - 遊戲流程說明
- **FR-006**: 系統必須提供可折疊/展開的規則說明功能，預設狀態為折疊
- **FR-007**: 系統必須顯示美術資源版權聲明區，包含花牌圖像來源聲明、開源授權資訊和第三方資源 attribution
- **FR-008**: 系統必須提供導航列，包含 Logo/遊戲名稱和導航連結（規則、關於、開始遊戲）
- **FR-009**: 使用者必須能夠透過導航列快速跳轉至對應區塊或頁面
- **FR-010**: 系統必須在小螢幕設備（如手機）上將導航列折疊為 hamburger menu
- **FR-011**: 系統必須在點擊導航列的「規則」連結時，平滑滾動至規則介紹區並自動展開規則說明
- **FR-012**: 系統必須在點擊導航列的「開始遊戲」連結時，導航至遊戲頁面
- **FR-013**: 系統必須防止使用者快速點擊多次「開始遊戲」按鈕導致重複導航
- **FR-014**: 系統必須支援鍵盤導航，所有互動元素可透過 Tab 鍵和 Enter 鍵操作（無障礙需求）

### Key Entities *(include if feature involves data)*

- **Hero Section**: 包含遊戲標題、副標題、主要 CTA 按鈕和視覺設計元素
- **規則說明內容**: 包含遊戲目標、牌組構成、牌的分類、基本役種介紹和遊戲流程說明，支援折疊/展開狀態
- **導航列**: 包含 Logo/遊戲名稱和導航連結（規則、關於、開始遊戲）
- **版權聲明內容**: 包含花牌圖像來源聲明、開源授權資訊和第三方資源 attribution

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 新手玩家在首頁停留時間達到 30 秒以上，表示成功吸引使用者閱讀內容
- **SC-002**: 首頁到遊戲頁面的轉換率達到 40% 以上，表示 Hero Section 和 CTA 有效傳達價值
- **SC-003**: 90% 的使用者能在首次訪問時成功找到並閱讀規則說明
- **SC-004**: 規則說明區塊的展開率達到 60% 以上，表示內容設計有效引起使用者興趣
- **SC-005**: 使用者在閱讀規則後能正確回答至少 3 個基本問題（遊戲目標、牌組構成、基本役種），正確率達 80% 以上
- **SC-006**: 頁面載入時間（First Contentful Paint）在 1.5 秒內，確保良好的首次體驗
- **SC-007**: 首頁在 Chrome、Firefox、Safari、Edge 四個瀏覽器上的視覺呈現一致性達到 95% 以上
- **SC-008**: 首頁在手機裝置（iOS、Android）上的可用性測試滿意度達到 4.2/5 以上
- **SC-009**: 使用者透過導航列跳轉至規則區塊的成功率達到 100%
- **SC-010**: 使用鍵盤導航的使用者能在 10 秒內完成「開始遊戲」操作

## Assumptions *(optional)*

- **A-001**: 使用者主要使用桌面瀏覽器（Chrome、Firefox、Safari、Edge）訪問首頁，移動端使用者佔比約 30%
- **A-002**: 使用者的網路連線速度為中等水準（4G 或以上），能在 3 秒內載入首頁
- **A-003**: 花牌圖像資源來自公有領域或開源授權（如 Wikimedia Commons、dotty-dev/Hanafuda-Louie-Recolor），無版權爭議
- **A-004**: 使用者主要語言為英文，規則說明和介面文字以英文為主
- **A-005**: 使用者對花牌遊戲的認知度較低，需要完整的規則介紹才能開始遊戲
- **A-006**: 遊戲頁面的路徑為 `/game` 或類似的 URL 結構
- **A-007**: 「關於」區塊包含專案背景、技術棧說明和開發者資訊
- **A-008**: 首頁不涉及使用者登入或帳號系統，任何訪客都能直接開始遊戲

## Dependencies & Integrations *(optional)*

- **D-001**: 依賴 Vue 3 + TypeScript 前端框架
- **D-002**: 依賴 Tailwind CSS 進行樣式設計
- **D-003**: 依賴花牌圖像資源（來自 Wikimedia Commons 或 dotty-dev/Hanafuda-Louie-Recolor）
- **D-004**: 依賴字體資源（Noto Serif、Inter、Noto Sans、Noto Sans JP）
- **D-005**: 首頁需要能夠導航至遊戲頁面（/game），依賴前端路由系統

## Scope & Limitations *(optional)*

### In Scope

- Hero Section（遊戲標題、副標題、主要 CTA 按鈕、視覺設計）
- 規則介紹區（基本規則說明、可折疊/展開功能）
- 美術資源版權聲明區
- 導航列（Logo、導航連結、響應式設計）
- 桌面和移動端的響應式設計
- 基本的無障礙支援（鍵盤導航）

### Out of Scope

- 使用者帳號系統或登入功能（MVP 階段不實作）
- 多語言支援（MVP 僅支援英文）
- 教學模式或互動式教學（Post-MVP）
- 深色模式（可選功能）
- 遊戲音效或背景音樂（Post-MVP）
- 進階無障礙功能（如螢幕閱讀器完整支援）
- 社交分享功能
- 遊戲統計或排行榜預覽
